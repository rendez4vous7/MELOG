# MELOG (Musical Engineering LOgic Graph)

**MELOG**는 타브 악보(TAB)의 6선 시스템을 데이터 카테고리로, 프렛 번호(Fret)를 논리적 상태로 치환하고, 정간보(Jeongganbo)의 격자를 절대 시간축으로 치환하여 세상을 좌표화하는 음악 기반 공학 데이터 언어입니다.

## 1. 철학 및 목적 (Philosophy and Purpose)
*   **언어 유형:** 논리적 엄밀함(Logic)과 정보의 압축(Experimental)을 동시에 지향하는 공학어.
*   **핵심 목적:** 연구 및 학술 데이터 전달 시 비모호성 극대화 및 정보 압축.

### [기본 원칙]
*   **제1원칙:** 모든 단어는 고유한 좌표(TAB)를 가지며, 모든 관계는 음악적 기호로 연산한다.
*   **제2원칙:** 텍스트형(Linear)은 사고의 기록을 위한 것이고, 악보형(Graphical)은 통찰의 공유를 위한 것이다.
*   **제3원칙:** 모든 숫자는 해당 도메인의 정의에 따라 '프렛' 또는 '수치'로 자동적으로 해석된다.

## 🌐 잠재적 활용 분야 (Potential Applications)
MELOG는 고도의 정밀도와 정보 압축이 필요한 다음과 같은 분야에서 혁신적인 데이터 인터페이스가 될 수 있습니다.

*   **로보틱스:** 복잡한 다축 관절의 상태 좌표 및 정밀 제어 시퀀스 기록
*   **자율주행 및 항공우주:** 초단위의 센서 데이터 스트림을 시각적 악보 형태로 압축 및 모니터링
*   **퀀트(금융):** 다변수 금융 파생상품의 상관관계를 리듬과 화성 구조로 분석
*   **국방 및 보안:** 비정형 데이터의 암호화 전송 및 실시간 전술 상황의 도식화
*   **의료 공학:** 생체 신호(EEG, ECG)의 파형을 논리적 상태값으로 변환하여 진단 보조

## 2. 체계 및 기호 정의 (System and Symbol Definitions)

### 2.1 악보형 (Graphical Mode)
데이터의 시각적 공간성과 동시성을 표현하는 시스템 로그 시트입니다.

*   **X축 (정간보 격자):** 절대 시간 클럭(Absolute Time Clock). 모든 칸은 물리적으로 동일한 가로 길이를 가지며, 이는 고정된 시간 단위를 의미한다. 한 칸 안의 좌표들은 병렬 처리됨.
*   **Y축 (6선 타브):** 데이터 도메인. 1번 줄부터 6번 줄까지 고정된 데이터 카테고리를 담당한다.
*   **데이터 노드 (Rhythmic Number):** 격자 내의 숫자는 상태(프렛)를 의미하며, 기둥(Stem)은 흐름을 제어한다.
    *   **상향 기둥 (Up-stem):** 해당 도메인의 출력 (Output/발신)
    *   **하향 기둥 (Down-stem):** 해당 도메인의 입력 (Input/수신)
    *   **관통 기둥 (Through-stem):** 데이터의 중계 (Relay) 및 실시간 처리
*   **기둥의 질감 (Stem Texture):** 
    *   **직선:** 확정 데이터 (Confirmed)
    *   **점선:** 추정 데이터 (Estimated)
    *   **지그재그:** 노이즈 및 불안정 데이터 (Noise/Unstable)
*   **기호 연산 (Symbolic Operations):**
    *   **이음줄 (Tie):** 이전 프레임의 상태값이 변하지 않고 그대로 계승됨.
    *   **슬러 (Slur):** 여러 좌표/프레임을 하나의 논리적 그룹으로 묶음.
    *   **셈여림 (f, p):** 데이터 신뢰도 및 가중치 ($f$: 강한 상관관계, $p$: 미약한 변수).
    *   **비브라토 (~~~):** 오차 허용 범위(Tolerance) 내 진동.

### 2.2 텍스트형 (Linear Mode)
악보의 좌표를 문자열로 치환하여 기록하며, AI 및 데이터 통신에 최적화되어 있습니다.

*   **좌표 표기:** `줄'프렛` (예시: `2'1`)
*   **입출력 표시:** `^`(출력/능동), `_`(입력/수동), `|`(중계/실시간 처리)
*   **단어 생성:** `자음(줄 번호) + 모음(프렛 번호)` (예: `NE`)
*   **다자릿수 표기:** 줄 번호 뒤의 프렛 번호는 자릿수 제한 없이 연속 기입한다.
*   **수치 읽기법:** 연속된 숫자는 십진법으로 읽되, MELOG의 고유 발음을 살려 읽는다.
*   **단위 선언 (Unit Declaration):** 3번 줄의 기본 단위는 도메인 선언 시 대괄호`[]`로 명시한다. 문장 내 단위 변경 시 수치 뒤에 소문자 영문 표준 약어를 부착한다.

#### 2.2.1 계층적 좌표 표기 (Hierarchical Notation)
*   **싱글 틱 ('):** 논리적 상태(프렛 번호)를 호출할 때 사용.
*   **콜론 (:):** 고유 ID(관절 번호, 센서 번호) 지정.
*   **콤마 (,):** 동일 도메인 내에서 여러 ID를 집합적으로 지정.
*   **복합 표기:** ID와 상태를 동시에 지정할 경우 `줄:ID'상태` 순으로 표기.

## 3. 형태론 및 발음 체계 (Morphology and Phonology)

### 줄 번호 (자음) - 데이터 카테고리
| 줄 번호 | 자음 | 본래의 카테고리 | 조음 특징 |
| :--- | :--- | :--- | :--- |
| 1번 줄 | **N (ㄴ)** | 시간 / 순서 / 빈도 | 치경 비음 |
| 2번 줄 | **T (ㄷ)** | 주체 / 대상 / 변수 | 치경 파열음 |
| 3번 줄 | **K (ㄱ)** | 수치 / 양 / 범위 | 연구개 파열음 |
| 4번 줄 | **M (ㅁ)** | 상태 / 속성 / 성질 | 양순 비음 |
| 5번 줄 | **L (ㄹ)** | 관계 / 논리 / 연산 | 유음 |
| 6번 줄 | **S (ㅅ)** | 환경 / 배경 / 도메인 | 치조 마찰음 |

### 프렛 번호 (모음) - 논리적 상태
| 프렛 | 발음 | 의미 (Meaning) |
| :--- | :--- | :--- |
| 0번 | **a (아)** | 중립 / 기점 (Neutral) |
| 1번 | **e (에)** | 긍정 / 내부 / 전진 |
| 2번 | **i (이)** | 부정 / 외부 / 후퇴 |
| 3번 | **o (오)** | 반복 / 집합 / 다수 |
| 4번 | **u (우)** | 심화 / 하단 / 고착 |
| 5번 | **oi (외)** | 확장 / 상단 / 변형 |
| 6번 | **ʌ (어)** | 이면 / 보조 / 대기 |
| 7번 | **ɯ (으)** | 수평 / 평형 / 보류 |
| 8번 | **y (위)** | 초점 / 강조 / 특수 |
| 9번 | **wa (와)** | 복합 / 완성 / 전체 |

## 4. 구문론 (Syntax)

### 4.0 도메인 선언 (Domain Declaration)
데이터 해석 분야를 최우선으로 선언합니다.

1.  **전역 선언 (Global Declaration):** 
    *   텍스트형: 최상단에 `$DOMAIN` 기입. 세션 종료(`||`)까지 유효.
    *   악보형: 첫 페이지 최상단 좌측에 명시.
2.  **지역 선언 (Local Override):** 
    *   텍스트형: 문장 내부 `[조표]` 직전 `(DOMAIN)` 소괄호 표기.
    *   악보형: 해당 마디(격자) 상단에 소괄호 표기.

### 4.1 문장 구조
*   **텍스트형:** `(지역도메인) [조표][자리표][시간/배경][주체^][동작][대상_][결론] ||`
*   **악보형:** 전역(좌측 상단), 지역(개별 격자 위).

### 4.2 주요 선언자 및 격 조사
*   **G> (High Clef):** 상태 선언 (정적 관찰 데이터).
*   **F> (Low Clef):** 동작 선언 (동적 변화 실행).
*   **^ / _ :** 주격(행위자/독립변수) / 목적격(대상/종속변수).
*   **|| (Double Barline):** 문장 종결.

### 4.3 논리 및 관계 연산자
*   `+` / `-` : 증가 / 감소.
*   `!` / `*` : 부정(중단) / 강조(임계점).
*   `?` : 의문/미정 (`??` 진동 오차, `~?` 추정치).
*   `~` / `:` : 연결(소속) / 값 할당(Assignment).
*   `&` / `/` : 동시 발생 / 선택적 병렬.
*   `->` : 인과 관계 (악보의 슬러에 대응).
*   `>>` / `<<` : 크레센도(가속) / 데크레센도(감속).

### 4.4 정밀 제어 표기
*   `\` : 미세 오차 보정.
*   `.` : 단일 펄스 실행.

### 4.5 수치 표현 및 발음 (Numeric Compression)
*   **P** (받침 -p): $10^1$
*   **V** (받침 -v): $10^2$
*   **B** (받침 -b): $10^3$ (보다 큰 자릿수는 기호 누적)
*   **r (르):** 소수점(.) 구분자

### 4.6 수치 분해 규칙
1. 천 단위(`B`)가 존재하면 무조건 `B`를 기준으로 분해한다.
2. 천 단위 미만일 경우 백 단위(`V`)를 우선한다.

### 4.7 데이터 도메인 해석 우선순위
| 줄 번호 | 기본 해석 원칙 (Default Interpretation) |
| :--- | :--- |
| 1, 4, 5 | 프렛 우선 해석 (Fret Priority) |
| 2, 6 | ID 우선 해석 (ID Priority) |
| 3 | 항상 물리적 수치로 해석 (Physical Value) |

### 4.8 ID 지정을 위한 발음 확장
2번 줄이나 6번 줄에서 특정 대상을 지칭할 때 발음법:
*   **규칙:** `줄 자음 + ID 프렛 모음 + 상태 프렛 모음`

## 5. 고급 제어 로직 (Advanced Control Logic)

### 5.1 조표 (Key Signature)
*   **Sharp (#):** 시스템 활성화 및 가중치 상향 모드.
*   **Flat (b):** 시스템 대기 및 보수적/하향 연산 모드.
*   **Natural (♮):** 이전 문맥의 모든 변수 초기화(Reset).

### 5.2 화성학적 충돌 (Harmonic Collision)
한 격자 내 상충 데이터 공존 시 **불협화음**으로 규정. 시스템은 이를 에러로 감지하거나 우선순위에 따라 상쇄 연산 수행.

## 🛠 기술 스택 및 개발 계획 (Tech Stack & Development)
현재 MELOG 전용 통합 개발 환경(IDE) 구축을 위해 다음과 같은 계획을 가지고 있습니다.

*   **언어:** Python 3.x
*   **프레임워크:** `PyQt6` (현재 학습 및 프로토타이핑 단계)
*   **목표:** 
    *   6선 타브 기반의 데이터 입력 인터페이스 구현
    *   텍스트형(Linear Mode)과 악보형(Graphical Mode)의 실시간 상호 변환 렌더링
    *   도메인별 사전(Dictionary) 자동 완성 기능

> **Note:** 현재 개발자는 Python을 활용한 기본 프로그래밍이 가능하나, `PyQt` 라이브러리는 새로 학습하며 적용 중인 단계입니다. IDE의 구조 설계나 GUI 구현 방식에 대한 숙련자분들의 조언과 코드 리뷰를 언제나 환영합니다!
